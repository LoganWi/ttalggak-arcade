# 배포 규칙 (Deployment Rules)

1. **절대 사용자 동의 없이 실서버(main 브랜치)에 임의로 배포하지 마세요.**
2. 사용자가 `dev` 브랜치에 저장해달라고 요청하거나, 코드 수정을 요청했을 때는 **오직 로컬 및 dev 브랜치까지만** 작업해야 합니다.
3. 실서버(main) 병합 및 배포는 사용자가 "배포해", "실서버에 올려", "main 반영해" 등 **명시적으로 Vercel 실서버 배포를 지시한 경우에만** 실행해야 합니다.
4. 배포 시에는 반드시 `.agents/workflows/deploy.md` 워크플로우를 참조하여 안전하게 진행하세요.
